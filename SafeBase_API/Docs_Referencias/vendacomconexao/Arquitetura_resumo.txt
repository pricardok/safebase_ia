## Fluxo de Primeiro Acesso ForГ§ado e InvalidaГ§ГЈo de SessГЈo
**Problema Resolvido**: Um novo usuГЎrio recebia uma senha, mas nГЈo era forГ§ado a trocГЎ-la, e tokens de acesso antigos continuavam vГЎlidos apГіs um novo login, criando uma falha de seguranГ§a.
#
**SoluГ§ГЈo Implementada**:
- **Flag de Primeiro Acesso**: Adicionada a coluna `must_change_password` na tabela `users`. Ao criar um novo usuГЎrio (via integraГ§ГЈo ou signup), esta flag Г© definida como `true`.
- **InvalidaГ§ГЈo de SessГЈo com `session_salt`**:

  - Este `salt` Г© incluГ­do no payload do `access_token`.

- **Fluxo de Login Inteligente**:

  - Se `true`, em vez de um token de acesso normal, retorna um `temp_token` com escopo restrito (`"scope": "force_password_change"`) e um status HTTP `210`.
  - **Nova Rota `POST /auth/force-password-change`**: Endpoint dedicado que aceita o `temp_token` e a nova senha, atualiza a senha do usuГЎrio e define `must_change_password` para `false`.
